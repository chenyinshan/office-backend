# 生成图片中中文乱码的原因与解决办法

## 一、原因说明

你在 Cursor 里用「生成图片」功能（或其它 AI 文生图，如 DALL·E）时，**图片里的汉字经常出现乱码或方框**，主要因为：

| 原因 | 说明 |
|------|------|
| **模型限制** | 文生图模型多数针对英文优化，没有内置中文字体或对 CJK 字形渲染支持不足。 |
| **文本渲染方式** | 模型是按“语义”生成像素，而不是真正“画字”，对非拉丁字符的字形预测容易出错。 |
| **无字体参数** | 当前 API 一般不能指定「用某款中文字体」渲染，无法从配置上解决。 |

所以这是 **AI 文生图本身的限制**，不是项目或 Cursor 设置能直接修好的。

## 二、可行解决办法

### 方案 1：图里只用英文，中文放在图外（推荐）

- 生成示意图时，**描述里要求所有图内文字用英文**（如 Client、Auth Server、JWT、Step 1…）。
- 在 Markdown / Word 里用**图注或表格**写中文说明，例如：
  - 图注：「图 1：OAuth2.0 + JWT 流程（客户端 → 授权服务器 → 资源服务器）」
  - 或图下加「英文–中文」对照表。

这样图不会乱码，信息又完整。

### 方案 2：用代码生成带中文的图（可完全避免乱码）

由**本地脚本**画图并指定**支持中文的字体**，这样汉字由系统字体渲染，不会乱码。

本项目已提供一个示例脚本：

- **脚本路径**：`scripts/gen_oauth2_jwt_diagram.py`
- **作用**：生成「OAuth2.0 + JWT 流程」示意图，图中文字为中文。
- **依赖**：`pip install matplotlib`
- **运行**（在项目根目录）：
  ```bash
  python scripts/gen_oauth2_jwt_diagram.py
  ```
- **输出**：`assets/oauth2_jwt_flow_diagram_cn.png`

脚本会优先使用系统里已有的中文字体（如 macOS 的 PingFang SC、Windows 的 Microsoft YaHei），无需额外装字体即可正常显示中文。

其他需要中文的示意图也可以照此方式：用 Python（matplotlib/Matplotlib）+ 中文字体，或 Mermaid 等支持中文的工具生成图，再导出为 PNG 使用。

### 方案 3：用 Mermaid 等图表语法 + 中文

在 Markdown 里用 Mermaid 画流程图/时序图，**文字直接写中文**，由支持 Mermaid 的查看器（如 GitHub、VS Code 插件、Typora）渲染，一般不会乱码。需要 PNG 时可用 Mermaid CLI 或在线工具导出为图片。

---

**总结**：  
- **乱码原因**：AI 文生图对 CJK 文字渲染支持不足，属于模型限制。  
- **解决**：图内用英文 + 图外中文说明；或改用代码/中文字体生成图（如本项目 `scripts/gen_oauth2_jwt_diagram.py`）。
