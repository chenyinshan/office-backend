# 长链接与短链接及交互逻辑

> 对应任职资格「长链接、短链接及交互逻辑」「服务端—客户端消息推送」，在 OA 项目中的体现说明。

---

## 一、概念简述

| 类型     | 含义           | 典型协议/方式        | 交互特点                     |
|----------|----------------|----------------------|------------------------------|
| **短连接** | 一次请求一次响应，结束后关闭（或复用 TCP 但逻辑上仍是一次请求） | HTTP/1.x REST、HTTPS | 无状态、服务端不主动推、客户端轮询或按需请求 |
| **长连接** | 连接建立后保持打开，服务端可多次推送数据 | WebSocket、SSE（Server-Sent Events） | 有状态（连接会话）、服务端可主动推、适合实时通知/大屏 |

- **短连接**：OA 中绝大部分接口都是 REST（如 `GET /api/notifications`、`POST /api/workflow/start`），每次请求-响应为一轮，无状态。
- **长连接**：OA 中通过 **SSE** 提供「服务端持续推送」示例（见下文），用于体现长连接及推送能力；站内通知若需「实时红点」也可复用该通道。

---

## 二、在 OA 项目中的体现

### 短连接（已全面使用）

- **Gateway → Portal → User/Workflow**：所有 REST 调用均为短连接。
- **前端**：`fetch`/`axios` 调用 `/api/**`，每次请求独立，无长连。

### 长连接（SSE + WebSocket 示例）

- **SSE 接口**：`GET /api/sse/notify`（需登录态）。
  - 行为：建立 SSE 长连接后，服务端按固定间隔发送心跳/事件，连接保持直到客户端关闭或超时。
  - 代码位置：`oa-portal` 中 `SseNotifyController`，使用 Spring `SseEmitter`。
- **WebSocket 接口**：`WS /ws/demo`。
  - 行为：连接建立后，服务端每 5 秒推送一次心跳，并回显客户端发送的消息（Demo）。
  - 代码位置：`oa-portal` 中 `WebSocketConfig` + `DemoWebSocketHandler`。
  - 前端可通过 `new WebSocket('ws://<host>/ws/demo')` 连接（本地 dev 由 Vite 代理到 8081）。

### 协议选型简要（SSE vs WebSocket）

| 对比项     | SSE                     | WebSocket        |
|------------|--------------------------|------------------|
| 方向       | 仅服务端 → 客户端        | 双向             |
| 协议       | 基于 HTTP，`Content-Type: text/event-stream` | 独立协议，需升级握手 |
| 实现       | `SseEmitter`，前端 `EventSource` | 需 STOMP 或原生 WS |
| 适用       | 以「服务端推送」为主（如通知、日志流） | 强交互、双向消息 |

OA 当前用 **SSE** 做长连接示例，实现简单且与现有 HTTP 网关兼容；若后续要做「双向实时聊天」可再上 WebSocket。

## 三、前端如何对接 SSE（长连接）

**方式一：演示页（推荐先体验）**  
登录 OA 后，侧栏点击 **「长/短连接演示」**，在页面上可直观看到：
- **短连接**：点击「发一次短连接请求」→ 请求 `GET /api/notifications/unread-count`，显示一次结果即结束。
- **长连接**：点击「建立长连接」→ 请求 `GET /api/sse/notify`（带 Token），下方实时追加服务端推送的事件（如 `connected`、`heartbeat`），可点击「断开长连接」关闭。

**方式二：代码对接**

```javascript
// 建立长连接
const es = new EventSource('/api/sse/notify', { withCredentials: true });

// 收到服务端推送
es.onmessage = (e) => {
  const data = JSON.parse(e.data || '{}');
  console.log('SSE message', data);
  // 例如：更新未读数、弹窗提示等
};

es.onerror = () => {
  // 断线后可选择重连
  es.close();
};
```

- 若网关/门户做了鉴权，需保证带 Cookie 或 Token（如通过 query 或 Cookie），`EventSource` 本身不设自定义 Header，可选用 `?token=xxx` 或 Cookie 方式。
- 关闭页面或离开页面时调用 `es.close()` 释放连接。

## 四、小结

- **短连接**：REST 请求-响应即短连接，OA 全链路已体现。
- **长连接**：通过 `GET /api/sse/notify` 的 SSE 示例体现，服务端持续推送，便于面试时讲解「长链接、短链接及交互逻辑」与「服务端—客户端消息推送」。
